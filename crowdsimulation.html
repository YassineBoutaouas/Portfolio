Realtime Crowd Simulation

This project compares different approaches to realtime crowd simulation. Specifically, the project focuses on comparing object-oriented approaches to data-oriented approaches. To accomplish this, the project will conduct a series of experiments using Unity's Entity Component System to simulate crowds of non-player characters.

In order to run the simulation the project makes use of the three principles proposed by Craig W. Reynolds (Alignment, Repulsion and Attraction) in the paper "Flocks, Herds, and Schools: A Distributed Behavioral Model". It further extends the behavior by embedding a pathfinding solution to provide single entities with a global understanding of the area of navigation.

Crowd simulation - Pathfinding combined with flocking behavior. Goal distribution not included yet.

Roadmap to improve the simulation

The project is still in development. However, the program is already capable to run a stable simulation that combines pathfinding with the three principles mentioned above.

Overall improvement of performance Using data-oriented programming and multi-threading to improve the overall performance.
Improvement of finding neighbors Using Octrees and/or compute shaders to improve finding neighbor instances.
Density-based pathfinding Implementing a density grid to provide further navigational intelligence.
